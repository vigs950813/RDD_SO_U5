5.2.1 Objetivos del software de E/S

Un concepto clave del diseño de software de E/S se conoce como independencia del dispositivo. Esto significa que debe ser posible escribir programas que puedan leer archivos de distintos dispositivos, como disquetes, discos duros o CD-ROM, sin necesidad de modificar los programas para cada tipo de dispositivo. Por ejemplo, al ejecutar un comando como:

sort <entrada> salida

El sistema debe permitir que las entradas provengan de un disco flexible, disco duro o teclado, y que las salidas se dirijan a un disco flexible, disco duro o pantalla. Es responsabilidad del sistema operativo resolver las diferencias entre estos dispositivos, aunque usen controladores en software muy distintos.

Un concepto relacionado con la independencia del dispositivo es el de nombres uniformes. Los nombres de archivos o dispositivos deben ser cadenas o números, sin depender del tipo de dispositivo. Por ejemplo, en sistemas UNIX, todos los discos se integran en la jerarquía del sistema de archivos, y un disco flexible puede montarse sobre un directorio como /usr/ast/respaldo. Esto permite que, al copiar un archivo a /usr/ast/respaldo/lunes, en realidad se copie en el disquete, sin necesidad de que el usuario distinga entre dispositivos. Así, todos los archivos y dispositivos se manejan uniformemente mediante nombres de trayectoria.

Manejo de errores es otro aspecto importante del software de E/S. Los errores deben manejarse lo más cerca posible del hardware. Por ejemplo:

  • Si un controlador detecta un error de lectura, debe intentar corregirlo automáticamente.
  • Si no lo logra, el controlador de software puede intentar repetir la operación.
  • Errores como los causados por partículas de polvo pueden ser transitorios y corregirse al repetir la operación.

Sólo si las capas inferiores no logran resolver el problema se informa a las capas superiores, permitiendo que muchos errores se gestionen de manera transparente sin afectar a los niveles superiores del sistema.

Otro aspecto clave del software de E/S es la distinción entre transferencias síncronas (por bloqueo) y asíncronas (controladas por interrupciones):

  • En la E/S física, las transferencias son generalmente asíncronas. La CPU inicia la operación y realiza otras tareas hasta recibir una interrupción.
  • Para los programas de usuario, las operaciones de E/S son más simples si son síncronas, es decir, si el programa se suspende automáticamente hasta que los datos estén disponibles.

El sistema operativo se encarga de abstraer estas diferencias, haciendo que las operaciones controladas por interrupciones parezcan síncronas para los programas de usuario.
