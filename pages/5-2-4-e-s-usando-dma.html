5.2.4 E/S usando DMA

El acceso directo a memoria (DMA) es una técnica avanzada de gestión de operaciones de entrada/salida (E/S) que permite transferir datos directamente entre dispositivos periféricos y la memoria principal, sin la necesidad de intervención continua de la CPU. Esta técnica aborda las limitaciones de la E/S programada y por interrupciones, optimizando el uso de los recursos del sistema.

En el modelo DMA, un módulo controlador asume el control del bus del sistema para realizar las transferencias de datos. Este módulo incluye registros especializados que gestionan los datos, la dirección y el conteo de palabras necesarias para las operaciones de E/S. La CPU inicializa el proceso configurando estos registros, especificando la dirección de memoria, el número de palabras a transferir y el tipo de operación (lectura o escritura). Posteriormente, la tarea se delega al módulo DMA, permitiendo que la CPU continúe con otras actividades.

Aspecto Descripción Ventajas Desventajas
Control del Bus El módulo DMA toma control del bus para realizar transferencias directas entre dispositivos y memoria. Elimina la intervención constante de la CPU, mejorando la eficiencia. Requiere arbitraje para evitar conflictos en el uso del bus.
Registros del DMA Incluyen registros de datos, dirección y contador de palabras para gestionar las transferencias. Permiten transferencias automáticas y precisas. Diseño más complejo del hardware del controlador.
Modos de Operación Diversos modos como 'robo de ciclos' o 'ráfaga' ajustan el uso del bus según las necesidades. Ofrecen flexibilidad para manejar diferentes volúmenes de datos. El modo 'ráfaga' puede dejar a la CPU inactiva durante períodos prolongados.
Interrupción Final El DMA genera una interrupción para notificar a la CPU al finalizar una operación. Minimiza la supervisión de la CPU durante la transferencia. Requiere un manejo eficiente de interrupciones en el sistema operativo.
Eficiencia en Grandes Transferencias Es ideal para mover grandes volúmenes de datos entre memoria y dispositivos. Incrementa significativamente el rendimiento del sistema. No es práctico para transferencias pequeñas o esporádicas.

En conclusión, el uso de DMA representa una solución eficiente para transferencias de datos en sistemas modernos, al permitir que la CPU se enfoque en otras tareas mientras el módulo DMA se encarga de la transferencia. Aunque su implementación requiere hardware especializado y un manejo adecuado de recursos, es una técnica fundamental para garantizar la eficiencia en sistemas con altas demandas de E/S.
