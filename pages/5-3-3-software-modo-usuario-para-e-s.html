5.3.3 Software modo usuario para E/S

El software modo usuario para E/S complementa el sistema operativo mediante bibliotecas y programas que operan fuera del núcleo (kernel). Aunque la mayor parte del software de E/S está integrada en el sistema operativo, algunas funciones esenciales se implementan en bibliotecas enlazadas a programas de usuario o como procesos independientes.

Bibliotecas de E/S

Las bibliotecas de E/S incluyen procedimientos que facilitan la interacción con el sistema operativo. Por ejemplo, una llamada como:

count = write(fd, buffer, nbytes);

en un programa en C, se resuelve mediante el procedimiento de biblioteca write, que se enlaza al programa en tiempo de ejecución. Estas bibliotecas colocan los parámetros en el lugar apropiado para la llamada al sistema y, en algunos casos, realizan tareas adicionales, como el formateo de entradas y salidas.

Un ejemplo de formateo es el uso de printf, que toma una cadena de formato y variables, construye una cadena ASCII y luego invoca write para enviarla a la salida. De manera similar, scanf lee entradas y las almacena en variables basadas en un formato específico. Estos procedimientos son parte de la biblioteca estándar de E/S y se ejecutan como parte de programas de usuario.

Sistemas de Spool

Además de las bibliotecas, el software de E/S incluye sistemas de spool, diseñados para manejar dispositivos de uso exclusivo en sistemas multiprogramados. Un ejemplo típico es el manejo de una impresora. Permitir el acceso directo a la impresora por parte de cualquier proceso podría causar problemas, como bloquear el dispositivo innecesariamente.

Para evitar esto, se utiliza un proceso especial llamado demonio y un directorio de spool. El flujo típico de trabajo es el siguiente:

  • El proceso usuario genera el archivo a imprimir y lo coloca en el directorio de spool.
  • El demonio, que es el único con permiso para acceder directamente al dispositivo, toma los archivos del directorio y los envía a la impresora.

Este enfoque asegura que ningún proceso usuario pueda monopolizar el dispositivo, garantizando un uso más eficiente y ordenado.
